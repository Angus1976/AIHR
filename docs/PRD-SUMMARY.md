# PRD v1.1 要点摘要（工程对齐用）

> 与聊天/外部 PRD 全文一致处以此摘要为**追溯入口**；细节以产品正式 PRD 为准。

## 产品定位

- **名称**：职得 / Jobde（支持白标：名称、Logo、主题色、欢迎语等）。
- **终端**：微信小程序（主）+ Web 管理端；风格统一，支持暗黑与无障碍。
- **价值**：AI 驱动全流程就业服务；结果导向收费；腾讯生态（会议、企微、支付、签名等）；线下代操作 + 伙伴入驻闭环。

## 角色与 MVP 行为


| 角色       | MVP 重点                           |
| -------- | -------------------------------- |
| 求职者      | 小程序侧核心 6 大环节 AI 服务、协议与付费节点       |
| 企业/HR    | **平台代发布**岗位、查看匹配列表、面试/入职反馈、简单仪表盘 |
| 就业老师/平台方 | 代录入、代操作、审核伙伴、运营配置                |
| 培训机构等伙伴  | 入驻提交资质与课程 → 平台审核上架               |
| 管理员      | AI 配置面板全权限；合规模板与审计               |


## 六大 AI 环节（逻辑模块）

PRD 要求模块化、可配置 LLM/提示词/Skills；MVP 优先核心环节，**不强调 AI/真人区隔**（人机混合可配置比例与时段）。

## 商业模式（工程需留痕）

- **求职者**：offer 后定金、第 2/3 月尾款、转正后赠送职后服务与可选续费（具体金额以 PRD 为准，需可配置）。
- **企业**：年费订阅档 + 成功入职佣金（比例分档、分两期支付）+ 增值与推广。
- **猎头/中介**：工具订阅 + 分成规则。
- **培训机构**：入驻分成、联合包、B2B 佣金。

→ 工程侧需要：**订单/账单/分账状态机**、**合同与签章关联**、**配置化定价**（避免硬编码）。

## AI 配置面板（Web 全功能 / 小程序精简）

- 模型路由、提示词库（版本/A/B）、Skills 编排、按环节绑定服务、日志与合规审计。
- 权限：管理员 CRUD；就业老师只读或有限编辑（按 PRD）。

## 合规（中国）

- PIPL：最小必要、单独同意、目的限定、敏感信息加强、审计与应急预案。
- 《电子签名法》：可靠电子签名（CA）、协议在线签、留痕。
- 数据存储本地化、跨境风险控制；后台合规模板可动态更新。

## 技术栈（PRD 未锁定）

PRD 允许 Vue/React 等。本仓库架构文档采用**中性描述**：前后端分离 + API 驱动配置热更新；具体语言在选型确定后写入 `docs/ARCHITECTURE.md` 的版本记录表。
